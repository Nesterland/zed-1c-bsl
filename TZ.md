## 1. Общие сведения

### 1.1 Наименование
**1C BSL** — расширение для IDE Zed с поддержкой 1C:Enterprise BSL, OneScript и SDBL.

### 1.2 Репозиторий
https://github.com/Nesterland/zed-1c-bsl

### 1.3 Текущая версия: 0.0.2 (в разработке)

### 1.4 Технологический стек

| Компонент | Технология |
|-----------|-----------|
| IDE | Zed |
| Парсер | Tree-sitter (alkoleft/tree-sitter-bsl, rev 5152121) |
| Подсветка | Tree-sitter queries (.scm) |
| LSP-сервер | bsl-language-server (Java, JVM) |
| Сниппеты | JSON Zed |
| Язык расширения | Rust (WASM) — опционально |

## 3. Фаза 3: indents.scm + базовая LSP (v0.0.3)

### 3.1 Модуль: авто-отступы (indents.scm)

**Файл:** languages/bsl/indents.scm | **Приоритет:** Высокий | **Трудоёмкость:** 2-4 часа

| ID | Требование | Статус |
|----|-----------|--------|
| IND-01 | Отступ тела Процедура/Функция — КонецПроцедуры/КонецФункции | ✅ |
| IND-02 | Отступ Если...Тогда — КонецЕсли | ✅ |
| IND-03 | Отступ циклов Для/Пока/Для каждого...КонецЦикла | ✅ |
| IND-04 | Отступ Попытка...Исключение — КонецПопытки | ✅ |
| IND-05 | Отступ #Область — #КонецОбласти | ✅ |
| IND-06 | Отступ #Если — #КонецЕсли | ✅ |
| IND-07 | Проверить все правила в реальном Zed | ⬜ |
| IND-08 | Сверить имена узлов Tree-sitter с логом zed: open log | ⬜ |
| IND-09 | Отступ для скобок ( и многострочных выражений | ⬜ |
| IND-10 | Outdent для Иначе/ИначеЕсли внутри Если...КонецЕсли | ⬜ |

**Критерии приёмки:**
- Открыть examples/basic.bsl в Zed
- Enter внутри блока → отступ увеличивается на 4 пробела
- Написать Конец... → отступ возвращается на уровень открывающего ключевого слова

### 3.2 Модуль: базовая LSP-интеграция

**Файлы:** extension.toml, languages/bsl/config.toml, docs/LSP_SETUP.md | **Приоритет:** Высокий | **Трудоёмкость:** 1-2 часа

#### Результаты анализа BSL Language Server

| Свойство | Значение |
|----------|---------|
| Исполняемый файл | bsl-language-server.jar (Java) |
| Команда запуска | java -Xmx2g -jar bsl-language-server.jar |
| Конфигурация | .bsl-language-server.json в корне проекта |
| Протокол | LSP через stdin/stdout |
| Возможности | Диагностика, форматирование, hover, go-to-def, references, rename, code actions, semantic tokens, folding, call hierarchy |
| Ограничение | Для авто-запуска нужен Rust WASM (Фаза 4) |

#### Требования

| ID | Требование | Статус |
|----|-----------|--------|
| LSP-01 | Задекларировать [language_servers.bsl] в extension.toml | ✅ |
| LSP-02 | Связать язык BSL с LSP в config.toml (language_servers = [bsl]) | ✅ |
| LSP-03 | Создать docs/LSP_SETUP.md с пошаговой инструкцией | ✅ |
| LSP-04 | Обновить README.md — добавить раздел про LSP | ✅ |
| LSP-05 | Создать шаблон .bsl-language-server.json | ✅ |

**Критерии приёмки:**
- После настройки settings.json — индикатор LSP для .bsl файлов
- Диагностики в examples/basic.bsl
- Автодополнение и hover работают
