# Аудит возможностей 1C BSL расширения для Zed

## Источники

- `https://github.com/1c-syntax/intellij-language-1c-` — ссылка возвращает 404.
- Корректный репозиторий: `https://github.com/1c-syntax/intellij-language-1c-bsl`.
- `https://infostart.ru/1c/articles/1789716/` — статья про настройку разработки 1С в VS Code с BSL Language Server.
- `https://github.com/1c-syntax/bsl-language-server` — основной Language Server Protocol сервер для BSL и OneScript.
- `https://github.com/alkoleft/tree-sitter-bsl` — Tree-sitter грамматика BSL/SDBL, которую использует текущее расширение Zed.

## Что уже учтено в текущем Zed-расширении

- Поддержка файлов `.bsl`.
- Поддержка файлов `.osl`.
- Поддержка файлов `.sdbl` для языка запросов 1С.
- Tree-sitter подсветка BSL.
- Tree-sitter подсветка SDBL.
- Подсветка встроенных SDBL-запросов внутри BSL-строк через `injections.scm`.
- Подсветка комментариев.
- Подсветка строк.
- Подсветка чисел.
- Подсветка дат.
- Подсветка процедур и функций.
- Подсветка вызовов методов.
- Подсветка аннотаций.
- Подсветка препроцессора.
- Поддержка скобок `()` и `[]`.
- Конфигурация автозакрытия `()`, `[]`, `""`.
- Сниппеты BSL.
- Пример BSL-файла.
- Пример SDBL-файла.
- Outline для процедур, функций и областей.
- Text objects для процедур, функций и комментариев.

## Что есть в Language 1C / BSL Language Server, но пока не реализовано в нашем Zed-расширении

- Подключение BSL Language Server как LSP.
- Диагностика кода BSL.
- Быстрые исправления / code actions.
- Форматирование файла.
- Форматирование выделенного диапазона.
- Go to definition.
- Find references.
- Rename symbol.
- Hover / подсказки.
- CodeLens с когнитивной и цикломатической сложностью.
- Inlay hints для параметров методов.
- Семантическая подсветка от LSP.
- Подсветка связанных конструкций `Если/Иначе/КонецЕсли`, циклов, `Попытка/Исключение/КонецПопытки`.
- Автоматические отступы по ключевым словам BSL через `indents.scm`.
- Автодобавление `|` внутри многострочных строк BSL.
- Автодобавление `//` при продолжении комментария.
- Запуск `.os`/`.bsl` через OneScript.
- Работа с метаданными конфигурации 1С.
- Исключение файлов на поддержке из анализа.
- Поддержка пользовательского `.bsl-language-server.json` описана, но не интегрирована в Zed UI.

## Приоритет следующих шагов

1. Проверить локальную установку текущего Zed-расширения.
2. Добавить `indents.scm` для базовых отступов BSL.
3. Добавить интеграцию BSL Language Server через Rust-код Zed extension.
4. Добавить инструкцию установки Java и BSL Language Server.
5. Добавить поддержку конфигурации `.bsl-language-server.json`.
6. Добавить semantic token rules для BSL Language Server.
7. Добавить runnables для запуска OneScript-файлов, если Zed позволит удобно передать команду.

## Вывод

Текущая версия — это уже полноценная базовая языковая поддержка Zed на уровне Tree-sitter: подсветка, injections, outline, text objects, сниппеты и примеры.

Но это ещё не полный аналог Language 1C для VS Code или IntelliJ-плагина. Главный недостающий блок — интеграция BSL Language Server.
